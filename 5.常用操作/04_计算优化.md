# 04 计算优化 - GPU加速

## 概要

GPU加速的关键是：设备管理一致性、张量与模型迁移、减少频繁数据拷贝。

---

## 设备管理

```python
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
```

便捷函数：

```python
def try_gpu(i=0):
    if torch.cuda.device_count() >= i + 1:
        return torch.device(f'cuda:{i}')
    return torch.device('cpu')
```

---

## 张量与设备

```python
x = torch.tensor([1, 2, 3])
x_gpu = x.to('cuda')
x_cpu = x_gpu.to('cpu')

y = torch.randn(3, 4, device='cuda:0')
```

要点：参与运算的张量必须在同一设备。

---

## 模型与GPU

```python
net = nn.Sequential(nn.Linear(10, 5), nn.ReLU(), nn.Linear(5, 1))
net.to(device)

X = torch.randn(100, 10, device=device)
y = torch.randn(100, 1, device=device)

optimizer = torch.optim.SGD(net.parameters(), lr=0.01)
criterion = nn.MSELoss()

for _ in range(3):
    pred = net(X)
    loss = criterion(pred, y)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

### 2. 计算精度与速度权衡

```python
# 混合精度训练（加快计算，节省内存）
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for epoch in range(num_epochs):
    for X, y in dataloader:
        # 自动选择低精度计算（如FP16）
        with autocast():
            y_pred = net(X)
            loss = criterion(y_pred, y)
        
        scaler.scale(loss).backward()
        scaler.step(optimizer)
        scaler.update()
```

### 3. 批处理大小优化

```python
# 批处理大小影响GPU内存使用和吞吐量
# 原则：尽量大但不超过显存限制

def find_max_batch_size(model, device, max_size=1024):
    for batch_size in [max_size, max_size//2, max_size//4, ...]:
        try:
            X = torch.randn(batch_size, 10, device=device)
            _ = model(X)
            return batch_size
        except RuntimeError:
            continue
```

## ⚠️ 常见问题

**Q: GPU加速能提升多少性能？**  
A: 取决于计算量和数据量。简单运算2-5倍，矩阵运算密集的场景10-100倍。

**Q: 为什么转移到GPU后反而变慢了？**  
A: 可能原因：数据量过小（并行开销大于加速），频繁的CPU↔GPU数据转移。

**Q: 多个GPU怎样充分利用？**  
A: 使用 `nn.DataParallel` 或分布式训练框架（如DDP）。

**Q: 是否可以在CPU上开发，GPU上训练？**  
A: 可以。只需将 `device = 'cpu'` 改为 `device = 'cuda'` 即可。

**Q: 什么时候考虑混合精度训练？**  
A: 当显存不足或需要加速时。但精度要求高的任务应谨慎使用。
