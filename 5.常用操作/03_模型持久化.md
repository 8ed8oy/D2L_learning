# 03 模型持久化 - 模型读写

## 概要

掌握张量保存、模型参数保存与加载，是训练和部署的基础。

---

## 张量读写

```python
x = torch.arange(4)
torch.save(x, 'x-file')

x2 = torch.load('x-file')

torch.save([x, x2], 'x-files')
xs = torch.load('x-files')

torch.save({'x': x, 'y': x2}, 'mydict')
mydict = torch.load('mydict')
```

---

## 模型参数保存与加载

```python
class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.hidden = nn.Linear(20, 256)
        self.output = nn.Linear(256, 10)

    def forward(self, x):
        return self.output(F.relu(self.hidden(x)))

net = MLP()
torch.save(net.state_dict(), 'mlp.params')

clone = MLP()
clone.load_state_dict(torch.load('mlp.params'))
```

要点：保存参数而非整个模型，更稳定、兼容性更好。

## 训练检查点

```python
checkpoint = {
    'epoch': epoch,
    'model_state': net.state_dict(),
    'optimizer_state': optimizer.state_dict(),
}
torch.save(checkpoint, 'checkpoint.pt')

loaded = torch.load('checkpoint.pt')
net.load_state_dict(loaded['model_state'])
optimizer.load_state_dict(loaded['optimizer_state'])
epoch = loaded['epoch']
```

### 3. GPU与CPU兼容性

```python
# 在GPU上保存，在CPU上加载
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')

# 保存前确保在正确设备上
net.to(device)
torch.save(net.state_dict(), 'model.pt')

# 加载到特定设备
net = MLP()
net.load_state_dict(torch.load('model.pt', map_location=device))
```

## ⚠️ 常见问题

**Q: state_dict() 包含什么？**  
A: 所有可训练参数（权重、偏置）和缓冲区（如BatchNorm的running_mean）。

**Q: 为什么不直接保存整个模型？**  
A: 容易出现版本兼容问题，文件较大。参数型存储更灵活。

**Q: 如何在不同框架间转换模型？**  
A: 使用ONNX格式进行跨框架转换，或使用TorchScript。

**Q: 保存后模型性能会变化吗？**  
A: 不会。只要正确加载，模型性能完全相同。
