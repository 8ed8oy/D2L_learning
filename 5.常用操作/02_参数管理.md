# 02 参数管理 - 访问、初始化、延迟初始化

## 概要

本节关注三件事：参数访问、参数初始化、延迟初始化。

---

## 参数访问

```python
net = nn.Sequential(nn.Linear(4, 8), nn.ReLU(), nn.Linear(8, 1))

# 单个参数
w = net[0].weight
b = net[0].bias

# 遍历所有参数
for name, param in net.named_parameters():
    print(name, param.shape)

# 参数字典
state = net.state_dict()
out_bias = state['2.bias']
```

要点：`named_parameters()`支持嵌套块；`state_dict()`适合保存与加载。

---

## 参数初始化

```python
nn.init.xavier_uniform_(net[0].weight)
nn.init.kaiming_normal_(net[0].weight, nonlinearity='relu')
nn.init.constant_(net[0].bias, 0)

def init_weights(m):
    if isinstance(m, nn.Linear):
        nn.init.normal_(m.weight, mean=0, std=0.01)
        nn.init.constant_(m.bias, 0)

net.apply(init_weights)
```

要点：Xavier适合tanh/sigmoid，Kaiming适合ReLU。

---

## 参数共享

```python
shared = nn.Linear(10, 5)
net = nn.Sequential(shared, nn.ReLU(), shared)
```

要点：多处引用同一层即可共享参数。

---

## 延迟初始化

```python
net = nn.Sequential(nn.Linear(10, 256), nn.ReLU(), nn.Linear(256, 10))
print(net[0].weight)  # 尚未初始化

X = torch.rand(2, 20)
_ = net(X)
print(net[0].weight.shape)
```

要点：第一次前向时自动推导输入维度并初始化参数。

# 训练循环
```
for X, y in data_loader:
    y_pred = model(X)
    loss = criterion(y_pred, y)
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()  # 使用初始化好的参数开始更新
```


## 常见问题

**Q: 为什么需要初始化参数？**  
A: 不同的初始化值会影响训练速度和效果。好的初始化能加快收敛，避免梯度消失/爆炸。

**Q: 所有参数都需要手动初始化吗？**  
A: 不需要。PyTorch有默认初始化。但在某些场景（如迁移学习）需要特殊初始化。

**Q: 延迟初始化会不会影响性能？**  
A: 性能影响可忽略，一般只在第一个forward时有微小开销。
